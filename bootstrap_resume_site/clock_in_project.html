MySQL Clock in Software

Placeholder

I created this project to practice various programming concepts. There are a lot of complex working parts that make up this application including SQL database connectivity, artificial intelligence algorithms, and GUI design. To create a modern graphical user interface, I designed a stationary main window that switches between tasks when buttons are clicked. The main screen plays a video clip until users register a new employee or starts the clock in software. Database connectivity is enabled through a File menu option, which prompts users to enter their MYSQL credentials when selected.


Placeholder

When registering a new employee into the system, a 'Take Photo' button is disabled until a face is detected inside of a specific screen region. To make this possible I used a simple cascade classifier trained to detect human faces. Once a face is found inside of the specified screen region (the white rectangle), users are able to take a photo which crops all pixels inside of it. This feature makes the application appear extremely modern and also prevents users from adding inaccurate image data into the system. After performing this action, the user is prompted a window to enter their name, which then adds the formatted photo into a folder for facial recognition.


Placeholder

Once users are added into the system, the main clock in option is enabled. This part of the program utilizes a facial recognition algorithm to detect all of the employees that have had their photo taken and labeled. I used a Python library called 'Facial Recognition' that allowed me to detect faces for all images added into a specific folder. When a face is detected on screen, the name of the employee is motion tracked with coordinates of the detected face and also enables the main 'Clock In' button. When a user clicks the button, a mini window updates showing the user the date and time of their clock in punch.


Placeholder

Clicking the 'Clock In' button also sends related data to the configured MYSQL database. The variables stored include a clock in ID (the primary key of the table), the employee's ID (obtained when registered), their name, the time that the employee clocked in at, and the date of the clock in. The data can be queried through MYSQL Workbench and simplifies many steps in traditional clock in applications. When an employee walks away from the clock in camera, all of the clock in data disappears from the display window. Code can be found on my Github.
